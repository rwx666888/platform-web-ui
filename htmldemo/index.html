平台前端框架使用手册

备注：

PS: iframe框架 使用 subindex.js 、inspinia.js 、 style.css

PS: 以下文件禁止直接替换官方原文件（源码有定制修改） ：datatimepicker.js(添加日期变更后自动触发表单验证功能，修正重置起始时间无效的问题) 、 dataTables.select.js(定制扩展)、 layer.js(定制扩展)、jquery.number.js（完全重写，但API:doc 与原组件保持一致）、jquery.validate.js（定制扩展）、select2.js(bug修正); dataTables.bootstrap.js(定制扩展，新增跳页解析器)、daterangepicker(定制扩展)、bootstrap-dropdown-on-hover-plugin（修正BUG）

PS: input 标签type="text"(不要使用 email，date等); 优先使用readonly="readonly"； 非必需慎用disabled="disabled"；

PS: 如果需要引入额外的公共组件，需要使用COM_TOOLS.requireJsFn(['需要加载的组件数组'],['所需加载组件所依赖的组件数组'],function(){ //TODO; }); 动态加载JS及CSS,并完成所加载组件的默认配置信息初始化，否则默认的定制化配置信息不会生效；

文件命名规则：demo页面：组件名-demo.html； demo数据文件或演示范例：以“demo组件名”开头[suf]

框架整体概述
PS：组件分为三类（基础、功能、框架），注意依赖关系；参照下方顺序使用：
js:
<!-- 基础组件 -->
<script src="jquery-2.1.1.js"></script>
<script src="bootstrap.min.js"></script>
<script src="jquery.metisMenu.js"></script><!-- 多级菜单 首页index【main】必须 -->
<script src="layer.js"></script><!-- 弹窗 -->
<!-- 功能组件 -->
<script src="jquery.dataTables.min.js"></script>
<script src="jquery.validate.custom.min.js"></script>
。。。。。。引入你需要的组件

<!-- 框架组件 -->
<script src="jquery.hash.min.js"></script><!-- 首页index【main】必须 -->
<script src="i18n/zh-CN.js"></script><!-- 国际化语言资源包（/private/i18n/XXXX.json）  2017-8-28（新增） -->
<script src="subindex.js" ></script><!-- 框架公共方法库 -->
<script src="inspinia.js"></script><!-- 首页index【main】必须 -->

CSS:
<!-- 基础组件 -->
<link href="bootstrap.min.css" rel="stylesheet">
<link href="font-awesome.css" rel="stylesheet"> (从2017-5-10版本后，为可选样式)
<!-- 功能组件 -->
<link href="plugins/iCheck/custom.css" rel="stylesheet">
。。。。。。引入你需要的组件

<!-- 框架组件 -->
<link href="animate.css" rel="stylesheet"><!-- 可选 -->

<link href="style.css" rel="stylesheet">

 * @param {Array} params 可选，替换模板中{n}的位置，n为params数组下标从0开始

 * 输出："只能输入3至6个字符！"
 */
帮助信息模块
(不建议对绑定了validate的input元素 使用此模式)
（无气泡） （有气泡） 
建议使用：帮助按钮（带气泡）形式；

/* 气泡提示信息使用方法： */
/* 使用方法 将需要添加提示信息的元素 class中 加入”js-helpmsg“，然后设置 data-helpmsg="msgkey"； msgkey = TEDU_MESSAGE[key]; 例如：*/
例1：
<button class="btn btn-info js-helpmsg" data-helpmsg="platform.common.label.help_admin">气泡提示信息</button>
例2：（input[type="text"]元素如需气泡形式展示，必须明确设置属性data-titlemsg="true"）,其中data-helpdata属性为可选属性用于替换模板中{n}的位置，多个参数用逗号（,）分割；（例如："platform.plugin.validate.rangelength":"只能输入{0}至{1}个字符！"）
<input type="text" class="form-control js-helpmsg" data-helpmsg="platform.plugin.validate.rangelength" data-helpdata="3,6" data-titlemsg="true"/>

/* 默认值提示信息使用方法 */
/* 只能应用于input[type="text"][placeholder]的元素 使用方法与气泡提示信息一致 */
例3：
<input type="text" class="form-control js-helpmsg" data-helpmsg="platform.plugin.validate.rangelength" data-helpdata="3,6"/>

/* 帮助按钮 （打开帮助页面[data-helpbtn="#yourkey"]或指定页面[data-helpbtn="url"]）; */
/* data-helpbtn属性值如果以“#”开头，则打开帮助页面，并跳转到 yourkey对应的资源位置；其它形式则认为是个url地址打开；*/
/* 所有帮助按钮打开的页面，无论什么形式，都统一指向到同一窗口中，所以只要该窗口存在，不会重复打开新窗口； */
例4：
<a class="tedufont tedu-icon133 cushelpbtn js-helpbtn" data-helpbtn="#3" ></a>（无气泡）
<a class="tedufont tedu-icon133 cushelpbtn js-helpbtn js-helpmsg" data-helpbtn="#3" data-helpmsg="platform.common.label.help_admin"></a>（有气泡）

/* 弹窗右上角帮助按钮使用方法，规则与上面的帮助按钮一致 【helpBtn用法同：data-helpbtn】*/
例5：
cumParentWinModal('弹窗名称','add.html',{
	helpBtn:'#2' //帮助页锚点, #+资源编码（id）
});
